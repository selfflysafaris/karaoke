ANTI-DOMINANCE RULES
Auto-balance spaces out repeat requests based on how many other singers are waiting — nobody gets penalised when the queue is quiet.

Queue size → rule:
• Under 3 songs → no restriction
• 3–5 songs → no back-to-back
• 6–9 songs → max 1 in every 3 slots
• 10+ songs → max 1 in every 5 slots

⚠ BACK-TO-BACK badge appears on cards that violate the current rule after a manual drag.

Drag cards to reorder manually at any time. Hit ⚖️ Rebalance to re-apply the rules.
🎵 SONG LIST MANAGEMENT
The singer app loads songs from songs-data.js — a JavaScript file in the same folder as the app.

To update the song list:
1. Run the Python extraction script on your Mac to generate a new songs-data.js
2. Tap ⬆️ Upload in the header — pick the file and push directly to GitHub
3. Tap 🎵 ↻ Song List — singers get the new list immediately

First time setup: Tap ⚙️ to enter your GitHub username, repo name and token.
Credentials are saved on this device — you only need to do this once.

File locations (GitHub Pages):
• songs-data.js → root of your GitHub repo (same folder as index.html)
• songs.json → same folder (offline fallback — upload both together)
• PDF original → keep locally on your Mac — never needed on the server
